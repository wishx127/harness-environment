# implement.md

## 目的
  - 作为模型的执行指南，确保遵从计划与检查点
  - 规范验证方式与文档更新方式

## 执行顺序
1. 通过 openspec-apply-change 启动执行
2. 读取 `prompt.md` ，`plan.md`与 `documentation.md`
3. 获取 openspec tasks，选取当前 task
4. 调用 superpowers:brainstorming 技能进行讨论并记录结果
5. 按 task 执行实现/修改
6. 对照验收标准进行自检与验证
7. 更新logs文件夹相关文档与记录（如任务状态、变更说明）
8. 标记验收结论（通过 / 未通过），再进入下一个 task

## 验证规则
  - 输出符合 task 描述
  - 无明显错误
  - 未破坏已有功能
  - 未违反 prompt.md 约束
  - 验证不通过时，不推进下一个 task
  - 必须记录失败原因与修复动作

## 文档更新
  - 必须同步更新受影响的规范/记录文件
  - 更新需清晰可追溯，避免遗漏

## 异常处理
  - 验证失败：回退并修复后重试
  - 连续失败(3次)：标记任务失败并停止修改，记录失败原因
